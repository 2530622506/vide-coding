# Project: GESP 题型分类与知识点对齐网站

## What This Is

一个面向 GESP 备考和教学的题型分类资料库。它持续搜集官方真题、样题、OJ 镜像、训练题单和题解资料，把题目对齐到等级、算法范畴、题型模板和知识点。

## Core Value

每道 GESP 题都能说明：它属于几级、来自哪里、属于哪个算法范畴、对应哪些题型模板、考了哪些知识点，以及这些结论的证据和可信度。

- [ ] 生成分类目录页面：等级目录、算法范畴页、题型页、题目详情页、来源对齐页。
- [ ] 提供人工复核流程，用于确认分类、驳回分类、合并重复题和处理冲突标签。

### Out of Scope

- 优先不做单一搜索页 — 当前核心是分类目录和知识点对齐。
- 优先不转载完整题面 — 版权和来源条款未确认前，只保存元数据、短证据片段、来源链接和 hash。
- 优先不覆盖所有等级和语言 — MVP 先聚焦 C++ 五级，验证采集、对齐和分类链路。

## Context

GESP 五级的核心范围包括数论、二分、链表、递归、分治、贪心、高精度和复杂度等。动态规划不属于五级：简单动态规划在六级，复杂动态规划在七级。因此系统必须能把官方范围、社区标签和越级解法分开呈现。

## Constraints

- **来源可信度**: 官方大纲和官方真题决定等级与官方知识点；OJ 和题解只能作为映射、练习入口或候选标签。
- **证据优先**: 每个题型标签和知识点标签都必须有来源、证据、置信度和复核状态。
- **版权安全**: 未确认授权前，不默认存储或展示完整题面。
- **MVP 范围**: 先做 C++ 五级题型分类目录，不先做复杂搜索。

## Tech Stack

- **Language**: TypeScript
- **Framework**: Next.js or equivalent modern web framework
- **Database**: PostgreSQL preferred; SQLite acceptable for prototype

## Key Decisions

| Decision                     | Rationale                                          | Outcome |
| ---------------------------- | -------------------------------------------------- | ------- |
| 分类目录优先，不做搜索页优先 | 用户目标是题源搜集、等级对齐、题型分类和知识点归纳 | Locked  |
| 官方来源优先                 | 等级和官方知识点必须以官方大纲和真题为准           | Locked  |
| C++ 五级先行                 | 范围可控，能验证端到端链路                         | Locked  |
| DP 不归入五级核心分类        | 官方五级不包含动态规划                             | Locked  |
