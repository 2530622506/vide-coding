# GESP C++ 题库扩充待办

当前题型分类网站已经具备 React 分类目录、Nest API、MySQL 读模型、题目详情展示，以及用户新增、修改、删除能力。题库扩充从公开 C++ 题源分批推进，仍保持官方来源优先和第三方来源待复核。

## 执行边界

- 只采集 GESP C++ 相关题目，不混入 Python、Scratch 或图形化编程。
- 官方来源仍然决定等级、考试场次和官方知识点范围。
- 洛谷、GitHub、OJ 镜像、题解站点只作为练习入口、题目映射、辅助标签或参考答案线索。
- 默认使用公开页面、无登录态、低频抓取；当前不使用洛谷账号。
- 后续如需登录洛谷，必须低频、可暂停、可恢复，并避免批量高频访问。

## 优先来源

- CCF GESP 官方真题、样题、认证大纲。
- `jonaslgtm/gesp-exam-questions` 的 C++ 目录。
- 洛谷公开 GESP 题库检索页和公开题面。
- zqiceberg、Hydro、CSPOJ、信创计划、东方博宜、羽润编程等已注册的公开镜像或题单。

## 入库要求

每道新增题目进入 MySQL 前必须具备以下结构化信息：

- 所属等级：`level`
- 算法范畴：`algorithm_domain`
- 题型模板：`problem_type`
- 知识点：`knowledge_points`
- 来源证据：`source_versions`
- 可信度与复核状态：`confidence`、`review_status`
- 参考答案：选择题使用官方答案；编程题优先使用可验证 OJ 入口或人工复核后的 C++ 参考解。
- 理解示例：中文摘要、解题步骤和中文注释。
- 选择题选项：保存 A/B/C/D 结构化文本，不只保存答案字母。
- 图片资产：保存 `asset_url`、`source_url`、`source_page`、`sha256`、`alt_text` 和归属题目，不默认把二进制图片塞进 MySQL。
- 编程题讲解：参考解必须是 C++，并带中文注释；未验证前只展示参考入口和待复核状态。
- AI 生成内容：如果各网站都没有可采集的答案、图片或讲解，可以生成学习辅助内容，但必须显式标注 AI 生成、提示甄别，并保留待复核状态。

## 后续流程

1. 用 `npm run plan:next-public-oj -- --limit 30` 生成下一批公开 OJ 补采清单。
2. 对清单内题目低频抓取公开题面、样例和图片，不使用登录态。
3. 对已抽取完整题面和样例的编程题生成带中文注释的 C++ 参考解，跑样例验证后写入 `ai_generated_sample_verified`。
4. 根据公开题面和样例通过解法修正算法范畴、题型和知识点，状态仍保持 `needs_review`。
5. 将新增或补齐后的题目写入 MySQL，并保留 JSON 导出备份。
6. 跑分类、答案指导、复核队列、数据库、API、CRUD 和前端构建验证。
7. 前端按等级、算法范畴、题型和知识点展示新增数据。
8. 持续优化分类：对低置信度标签、缺失选项、缺失图片和缺失参考解建立复核队列，人工确认后回写 MySQL。

## 复核闭环

当前大量记录仍保留 `needs_review`，这是为了避免把第三方题源、AI 参考解或推断标签误认为官方结论。后续必须持续收敛复核队列：

- 用 `npm run plan:review-workqueue` 生成统一复核计划。
- 当前复核计划输出到 `data/classification/review-workqueue-plan.json`。
- 当前待复核工作项：1352 项，其中 high 74 项、medium 630 项、low 648 项。
- 优先处理 high：来源冲突、未补题面、缺 C++ 参考解。
- 再处理 medium：低置信度标签、补充题标签、选项/图片/样例缺口。
- 人工确认后再把 `needs_review` 提升到更高可信状态；AI 生成内容即使样例通过，也不能自动标成官方答案。
